10 /**
11  * This header defines the public api of the MiniLua library.
12  *
13  * We use the PImpl technique to hide implementation details (see below).
14  *
15  * TODO some more documentation on where to start and design decisions
16  *
17  * ---
18  *
19  * PImpl technique:
20  *
21  * Also see: https://en.cppreference.com/w/cpp/language/pimpl
22  *
23  * With the PImpl technique we hide all behaviour using a private nested forward
24  * declaration of a struct or class. This has two important uses:
25  *
26  * 1. hiding the implementation details from the user
27  * 2. breaking up cycles of incomplete types
28  * (this works because the cyclic reference are now in the cpp instead of the header)
29  *
30  * For this to work classes using the PImpl technique can't *use* the private nested
31  * forward declaration. This means we have to declare but not define the methods
32  * in the header. This includes special member functions like copy-constructor and
33  * destructor. Then these methods have to be implemented in the cpp file and the
34  * only difference is basically instead of using `this->` they have to use
35  * `this->impl->` or just `impl->`.
36  *
37  * A class with the PImpl technique usually looks like this:
38  *
39  * ```
40  * // in the header
41  * class Something {
42  * struct Impl;
43  * owning_ptr<Impl> impl; // any pointer type is ok here
44  *
45  * public:
46  * // normal constructor declarations
47  * Something(const Something&);
48  * Something(Something&&) noexcept;
49  * Something& operator=(const Something& other);
50  * Something& operator=(Something&& other);
51  * friend void swap(Something& self, Something& other);
52  * };
53  *
54  * // in the cpp
55  * struct Something::Impl {
56  * // fields you would have put in class Something
57  * };
58  * Something::Something(const Something&) = default;
59  * Something(Something&&) noexcept = default;
60  * Something& operator=(const Something& other) = default;
61  * Something& operator=(Something&& other) = default;
62  * void swap(Something& self, Something& other) {
63  * std::swap(self.impl, other.impl);
64  * }
65  * ```
66  *
67  * If the nested `struct Impl` is not default constructible you have to provide
68  * the implementation of the copy-/move-constructors and -operators manually and
69  * can't use `= default`.
70  *
71  * `owning_ptr<T>` was chosen for the pointer type because it behaves like a
72  * normal `T` (but lives on the heap). It's move, copy and lifetime semantics
73  * are identical to the one of `T`.
74  *
75  * It is also possible to choose another pointer type like `std::unique_ptr` or
76  * `std::shared_ptr`. You should probably avoid using raw pointer `T*` because
77  * lifetime management is harder with raw pointers.
78  */
