Detailed Description

Stores the state needed to execute a query and iteratively search for matches.

You first have to call 'exec' with the query and then you can retrieve matches with the other functions.

You can iterate over the result matches by calling 'next_match'. This is only useful if you provided multiple patterns.

You can also iterate over the captures if you don't care which patterns matched.

At any point you can call 'exec' again and start using the cursor with another query.

Features not include (because we currently don't use them):

  • setting byte/point range to search in:
    • ts_query_cursor_set_byte_range
    • ts_query_cursor_set_point_range
